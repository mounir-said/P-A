Short Answer Questions

1. What is source code?

Source code is the human-readable set of instructions written by programmers in a programming language. It is the fundamental component that gets translated into machine code to be executed by a computer.

2. What makes computers so “smart”, anyway?

Computers are considered "smart" because they can process vast amounts of data quickly, execute complex algorithms, and perform tasks with precision based on the instructions (programs) they receive from humans.

3. What is the purpose of a programming language?

The purpose of a programming language is to provide a way for humans to communicate with computers, giving them instructions in a form that can be translated into machine code for execution.

4. What are 3 examples of programming languages? Why are there so many of these?

Examples: JavaScript, Python, Java. There are many programming languages because different languages are designed to solve different types of problems, provide various levels of abstraction, and support different styles of programming (e.g., object-oriented, functional).

5. What is a variable? Why are variables useful?

A variable is a container for storing data that can change over time. Variables are useful because they allow programmers to store, manipulate, and reuse data throughout a program.

6. What is the difference between a single-equals (=) and a double-equals (==)?

A single-equals (=) is used for assignment, meaning it assigns a value to a variable. A double-equals (==) is used for comparison, checking if two values are equal, with type conversion.

7. What is the difference between a double-equals (==) and a triple-equals (===)?

Double-equals (==) checks for equality after type conversion, while triple-equals (===) checks for both value and type equality without converting types.

8. Why does the developer console exist?

The developer console exists to help developers debug their code by providing real-time feedback, error messages, and the ability to interact with the browser's JavaScript environment.

9. When we talk about “conditional” statements, what does that mean? What is an example?

Conditional statements are used to perform different actions based on whether a condition is true or false. Example: if (x > 10) { console.log('x is greater than 10'); }

10. Why would we want FOR or WHILE loops in our source code?

Loops allow us to repeat actions or iterate over data structures without manually writing the same code multiple times. They make code more efficient and easier to maintain.

11. When would you use a WHILE loop, instead of a FOR loop?

A WHILE loop is typically used when the number of iterations is not known beforehand, while a FOR loop is used when the number of iterations is predetermined.

12. What is a function? Why would we use functions?

A function is a block of code designed to perform a specific task. Functions are used to organize code, make it reusable, and improve readability.

13. How many values can you receive back from a function? How many values can you send in?

A function can return a single value. However, you can send in multiple values as arguments to a function.

14. What is an array? How many values does it hold?

An array is a data structure that can hold multiple values in a single variable. The number of values an array can hold is determined by its length.

15. What is a T-diagram and why should I know how to use one?

A T-diagram is a visual tool used to represent the input and output of functions. It helps in understanding how data flows through a program and aids in debugging.

16. What are the two ways to comment JS code? When would you use one versus the other?

There are two ways to comment JavaScript code: single-line comments using // and multi-line comments using /* comment */. Use single-line comments for brief explanations and multi-line comments for longer descriptions or to temporarily disable blocks of code.
