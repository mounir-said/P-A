Short Answer Questions

1. What is an array?

An array is a data structure that holds a collection of elements in a specific order. Each element is stored at a specific index, allowing for efficient access and modification.

2. What types of values can be held in an array?

Arrays can hold any data type, such as numbers, strings, booleans, objects, functions, and even other arrays.

3. What happens if you try to contain values of different data types in an array?

In JavaScript, arrays are dynamic and can contain values of different data types without issues. For example, an array can hold both strings and numbers simultaneously.

4. What is the index of the first element in an array?

The index of the first element in an array is 0.

5. What is the index of the last element in an array?

The index of the last element is array.length - 1.

6. If you control an array, what is the easiest way to determine the number of values?

The easiest way to determine the number of values is by using the length property of the array, which returns the total number of elements.

7. When would the above method not work well, and what must you do in that case?

The length property may not work well if the array has gaps (sparse arrays). In such cases, you must iterate through the array to count only the defined elements.

8. What are two ways to make an array one element longer?

1. Use the push() method to append an element.
2. Assign a value at the next index, e.g., arr[arr.length] = value;.

9. What are two ways to make an array two elements shorter?

1. Use the pop() method twice.
2. Use the splice() method to remove the last two elements, e.g., arr.splice(-2, 2);.

10. What is the upper limit on the number of values an array can hold?

JavaScript arrays are limited by the maximum length, which is 2^32 - 1 (approximately 4.29 billion elements).

11. When working with arrays, does working “in-place” really matter? If so, when and why?

Yes, working "in-place" can matter for memory efficiency. It is important when working with large arrays or performance-sensitive applications because creating new arrays for every operation may use more memory and slow down performance.

12. Which values are “falsy” in JavaScript, and what does “falsy” mean anyway?

Falsy values are values that are considered false when evaluated in a Boolean context. The falsy values in JavaScript are: false, 0, "" (empty string), null, undefined, and NaN. "Falsy" means that these values will return false when used in conditions or Boolean expressions.

13. What does “passing by reference” mean?

"Passing by reference" means that when an object (such as an array or another complex data type) is passed to a function, the function receives a reference to the original object. Changes made to the object inside the function will affect the original object.
